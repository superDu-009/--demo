## 【严重问题（必须改）】
### 1. 开放重定向漏洞
- 文件路径：`/src/views/Login.vue`
- 问题描述：第117行直接从URL query参数获取`redirect`并跳转，未校验是否为站内路径，攻击者可构造类似`https://your-domain/login?redirect=https://evil.com`的恶意链接，用户登录后会被跳转至钓鱼网站，窃取账号信息。
- 改进建议：跳转前校验`redirect`必须为站内相对路径（需满足以`/`开头且不包含`//`、`http://`、`https://`等外部协议前缀），仅允许站内跳转。
【小欧回复】：接受修改，计划1小时内完成：在跳转前增加路径校验逻辑，仅允许以`/`开头且不包含外部协议前缀的站内路径跳转，外部路径直接跳转到项目列表页。

### 2. 上传无限递归栈溢出风险
- 文件路径：`/src/composables/useTosUpload.ts`
- 问题描述：第82-84行预签名过期错误（40005）时直接递归调用`upload`方法，无重试次数限制，若后端持续返回40005错误会导致栈溢出、页面崩溃。
- 改进建议：增加最大重试次数限制（建议最多3次），避免无限递归。
【小欧回复】：接受修改，计划1小时内完成：新增重试次数计数器，最多重试3次，超过则抛出错误终止上传，避免无限递归。

### 3. 上传文件类型校验绕过风险
- 文件路径：`/src/composables/useTosUpload.ts`
- 问题描述：第34行仅通过浏览器提供的`file.type`属性校验文件类型，该属性可通过篡改文件后缀轻易绕过，存在上传恶意文件（如exe、webshell等）的风险。
- 改进建议：前端增加文件头（魔数）校验逻辑，同时明确要求后端必须做服务端的文件类型、大小校验，不可信任前端校验结果。
【小欧回复】：接受修改，计划2小时内完成：新增文件魔数校验函数，读取文件前几个字节校验真实文件类型，同时在代码注释和文档中明确要求后端必须做二次校验，不可信任前端校验结果。

### 4. XSS潜在风险
- 文件路径：`/src/components/Common/TosUpload.vue`
- 问题描述：第188-190行仅通过URL后缀判断是否为图片，若用户上传包含恶意脚本的SVG文件或伪装成图片的HTML文件，直接渲染可能触发XSS攻击。
- 改进建议：预览图片时增加Content-Type校验，禁止直接渲染SVG/HTML等可执行脚本的文件类型，或配置CSP策略限制脚本执行。
【小欧回复】：接受修改，计划1小时内完成：修改图片类型判断逻辑，仅允许`png/jpg/jpeg/gif/webp`等纯图片格式预览，SVG/HTML等类型仅显示文件图标，不渲染内容。

## 【一般问题（建议改）】
### 1. 401跳转死循环风险
- 文件路径：`/src/api/index.ts`
- 问题描述：第53行收到401未登录错误时直接跳转登录页，未判断当前是否已经处于登录页，可能导致重复跳转死循环。
- 改进建议：跳转前判断当前路由名称是否为`Login`，避免重复跳转。
【小欧回复】：接受修改，计划1小时内完成：在401拦截逻辑中增加当前路由判断，若已经处于登录页则不重复跳转，直接清除登录态即可。

### 2. 上传请求内存泄漏风险
- 文件路径：`/src/composables/useTosUpload.ts`
- 问题描述：`uploadToTos`方法创建的XHR请求未提供取消机制，若组件在上传过程中销毁，请求仍会继续执行，回调执行时可能导致报错或内存泄漏。
- 改进建议：在composable中提供`abort`方法，组件卸载时主动取消未完成的上传请求。
【小欧回复】：接受修改，计划2小时内完成：在useTosUpload中维护XHR实例，导出abort方法，TosUpload组件在onUnmounted时调用abort取消未完成的上传，避免内存泄漏。

### 3. 上传complete接口重试无延迟
- 文件路径：`/src/composables/useTosUpload.ts`
- 问题描述：第65-71行complete接口失败后立即重试，无延迟机制，瞬间重复请求可能增加后端压力。
- 改进建议：重试前增加1000ms左右的延迟，避免频繁请求。
【小欧回复】：接受修改，计划1小时内完成：重试前增加1000ms延时，降低后端请求压力。
