## 4. 核心页面设计

### 4.0 UI 视觉与组件策略

目标：
- 整体风格定位为“未来影视控制舱”，不是传统管理后台。
- 页面要同时满足好看、实用、科技感强三个目标：高频操作优先保证可读性和稳定性，视觉表达通过布局层、卡片层、背景层、状态层增强。

视觉规范：
- 背景：深空黑基底，叠加青色/金色径向光、网格纹理、轻扫描线。
- 主色：电弧青 `#5cf1ff`；辅助色：能量绿 `#7dffb2`、警示金 `#ffcc66`、异常粉 `#ff4fd8`。
- 字体：标题优先使用 `Rajdhani / DIN Condensed / Agency FB` 等科技窄体回退；正文使用 `Saira / PingFang SC / Microsoft YaHei`。
- 卡片：统一使用半透明毛玻璃、发光边框、斜切角、弱高光层。
- 状态：生成、解析、资产状态以标签和数据芯片呈现，避免只依赖颜色。
- 动效：保留轻量扫描线、悬浮上移、边框发光，避免高频操作区出现干扰性动画。

组件策略：
- viog-ui：用于主按钮、品牌化操作入口、后续可扩展为导航、卡片、标签、加载等科技组件。
- Element Plus：保留在表单校验、弹窗、下拉、上传、消息提示、分页等成熟交互，降低业务回归风险。
- 自研 HUD 类：`card-glass`、`hud-panel`、`hud-corner`、`scanline`、`data-chip`、`hud-title`、`hud-kicker` 作为跨页面视觉基座。
- 禁止只通过“紫色渐变 + 暗色背景”定义科技感，必须在信息层级、空间感、状态表达和动效上形成完整系统。

### 4.1 登录页 / 登录弹窗

功能：
- 仅支持用户名 + 密码登录。
- 登录成功提示文案统一为 `欢迎你，{用户名}~`。
- 页面登录和 token 过期弹窗登录复用同一接口协议。

实现说明：
- 登录页负责首次进入系统。
- `AppLayout.vue` 监听 `auth:expired` 事件，在任意业务页弹出登录弹窗，登录后留在当前页面。
- 登录页视觉采用“生产矩阵入口”：大标题、能力芯片、斜切角登录卡片、扫描线动效，主按钮使用 viog-ui 风格。

### 4.2 项目列表页

功能：
- 搜索项目名称。
- 新建项目：项目名称、小说文件、默认比例、默认清晰度、风格。
- 编辑项目：允许改名称、描述、默认比例、默认清晰度、自定义风格描述；已创建项目风格不可改。
- 删除项目：走软删接口。

约束：
- 所有可选项统一使用下拉框枚举：
  - 比例：16:8 / 16:9 / 4:3 / 9:16
  - 清晰度：720P / 1080P / 2K / 4K
  - 风格：2D次元风 / 日漫风 / 国漫风 / 古风 / 现代写实 / 自定义

视觉：
- 顶部为项目舰队 HUD 面板，展示项目数、已上传剧本数。
- 项目卡片保留封面图，但增加暗场渐变、OPEN 标识、数据芯片式元信息，提高可扫读性。

### 4.3 项目详情页

结构：
- 顶部展示项目名称、描述、默认比例、默认清晰度、风格。
- Tab 固定顺序：
  1. 剧本预览
  2. 分镜工作台
  3. 资产库

视觉：
- 项目概览使用 HUD 面板，默认比例/清晰度/风格以独立元信息卡展示。
- Tab 使用斜切角胶囊按钮，当前 Tab 使用电弧青到能量绿渐变高亮。

### 4.4 剧本预览页

功能：
- 原始剧本文本只读展示。
- 每页 500 字分页。
- 底部左右箭头翻页。
- 右上角「剧本分析」按钮弹窗，拆分规则以模板 key 枚举，不在前端硬编码 prompt 文本。
- 解析状态：
  - 待解析
  - 解析中
  - 解析成功
  - 解析失败
- 失败态展示错误原因与手动重试按钮。

当前实现说明：
- 读取项目详情返回的 `novelTosPath` 纯文本可读地址，前端直接分页展示。
- 点击「剧本分析」后调用 `/api/project/{projectId}/episodes/analyze` 提交任务。
- 前端轮询 `/api/project/{projectId}/episodes/analyze/status`，展示 `待解析 / 解析中 / 解析成功 / 解析失败` 与失败原因。
- `txt / md / docx` 的纯文本抽取统一由后端完成，前端不再自行解析 Word。
- 阅读器区域使用低干扰暗场纸面，保留 500 字分页的阅读舒适度，同时使用扫描线和边框光效强调控制舱语义。

### 4.5 分镜工作台

目标口径：
- 用户只看到“分集 → 分镜”，不展示分场。
- 左侧为分集列表；分集可展开查看完整内容与关联资产名称。
- 每个分集带「分镜拆分/重新拆分」按钮；已存在分镜时先弹二次确认。

右侧分镜卡片：
- 字段：中文提示词、英文提示词、景别、运镜、台词、时长、绑定资产、承接上一分镜开关、草稿内容、图片预览、视频预览。
- 支持新增、编辑、删除分镜。
- 支持批量生成提示词、批量生成图片、批量生成视频。
- 承接上一分镜默认开启，视频生成成功后自动记录尾帧占位值。

统一生成按钮规则：
- 未生成：`生成`
- 生成成功：`重新生成`
- 生成失败：`重试`
- 生成中：按钮 loading 且禁点

参数确认规则：
- 当前前端按项目默认比例与清晰度发起生成。
- 当视频或图片生成参数与项目默认值不一致时，弹出确认框：
  `当前参数与全局配置不一致是否确定生成`
- 若当前没有传入覆盖参数，视为使用项目默认参数，不弹出不一致确认。

接口说明：
- 分镜列表、拆分、新增均按 `episodeId` 对接，不再保留 `sceneId` 兼容层。
- 单个/批量生成统一返回 `taskId` 或 `batchId + taskIds`，前端按任务状态轮询刷新分镜列表。

### 4.6 资产库

结构：
- 顶部分类固定顺序：角色 → 场景 → 道具 → 声音。
- 顶部操作区：`提取资产`、`新建资产`、搜索。
- 卡片展示：
  - 缩略图
  - 名称
  - 类型
  - 状态
  - 父资产名称
  - 子资产缩略图条
  - `生成` / `编辑` / `删除`

功能：
- 提取资产：弹窗选择分集，调用提取任务接口。
- 重复资产检测：通过 `/api/project/{projectId}/assets/duplicates` 获取重复资产分组，弹出确认处理方式：
  - 合并
  - 删除
  - 保留
- 新建/编辑资产：
  - 资产类型下拉
  - 参考图上传
  - 是否子资产
  - 父资产多选
  - 草稿内容
- 生成按钮按统一按钮状态规则执行。
- 删除前查询引用；若资产已被分镜绑定则禁止删除。
- 资产生图按钮按统一按钮状态展示；如果后端资产生图接口不可用，前端以提示反馈，不阻塞资产库其他能力。

### 4.7 个人中心

入口：
- 右上角头像下拉菜单。

功能：
- 上传头像
- 修改用户名
- 修改密码
- 退出登录

实现说明：
- 用户名、头像、密码分别调用独立接口更新。
- 登录失效后在当前页弹窗重新登录，不跳转。

### 5.1 已接入接口

| 模块 | 接口 |
|------|------|
| 用户 | `/api/user/login` `/api/user/logout` `/api/user/info` `/api/user/username` `/api/user/password` `/api/user/avatar` |

| 任务 | `/api/task/{id}` `/api/task/batch/status` |

### 5.2 当前前端对接策略

1. 前端完全按项目 → 分集 → 分镜结构对接，不再保留分场兼容层。
2. 所有异步动作统一走任务接口轮询，包括剧本分析、分镜拆分、提示词/图片/视频生成、资产提取、资产图生成。
3. prompt 模板不在前端硬编码具体内容，只保留模板 key 枚举。
4. 小说上传仍走 TOS 预签名直传，创建项目时提交 `novelFile.fileName/fileKey/fileSize`；项目详情读取后端返回的纯文本可读地址。
